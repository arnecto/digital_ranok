# Новинний бот — інструкція із запуску

## 1. Створити Telegram-бота
1. Відкрий Telegram, знайди **@BotFather**.
2. Введи `/newbot`, дай боту ім'я і юзернейм (у тебе вже є: **@digital_ranok_bot**).
3. BotFather дасть токен виду `123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxx` — це `TELEGRAM_BOT_TOKEN`.

## 2. Створити канал і додати бота
1. Створи новий Telegram-канал (публічний або приватний) — наприклад **Цифровий Ранок** (`@digital_ranok`).
2. Зайди в налаштування каналу → **Адміністратори** → додай **@digital_ranok_bot**, дай право **публікувати повідомлення**.

## 3. Дізнатись chat_id каналу і власний chat_id
**Канал:**
- Якщо канал **публічний** — `TELEGRAM_CHAT_ID` = `@назва_каналу` (наприклад `@digital_ranok`).
- Якщо **приватний**:
  1. Опублікуй у каналі будь-яке повідомлення.
  2. Перешли (forward) це повідомлення боту **@userinfobot** або відкрий у браузері:
     `https://api.telegram.org/bot<TOKEN>/getUpdates`
  3. Знайди в JSON поле `"chat":{"id": -100xxxxxxxxxx, ...}` — це і є chat_id.

**Твій особистий chat_id** (щоб бот присилав тобі звіт про публікацію в особисті):
1. Знайди свого бота **@digital_ranok_bot** в Telegram і натисни **Start** (обов'язково — бот не може написати першим, поки ти йому не написав хоч раз).
2. Перешли (forward) будь-яке своє повідомлення з чату з ботом — боту **@userinfobot**, він покаже твій `id` (число без мінуса, наприклад `123456789`).
3. Це і є `TELEGRAM_ADMIN_CHAT_ID`.

## 4. (Опційно) Ключ Google Gemini API — безкоштовно
Якщо хочеш, щоб бот сам відбирав найважливіші новини і писав короткі summary українською:
1. Зайди на **https://aistudio.google.com/apikey**.
2. Увійди Google-акаунтом → **Create API key** (картка не потрібна).
3. Скопіюй ключ → це `GEMINI_API_KEY`.

Скрипт використовує модель-псевдонім `gemini-flash-lite-latest` — Google сам оновлює, на яку конкретну модель він вказує, тож скрипт не зламається, коли стару модель знімуть з підтримки (як уже сталось раз).

Без цього ключа бот просто публікує перші 5 новин з кожної категорії без AI-фільтрації.

## 5. Встановити залежності
```bash
pip install -r requirements.txt
```

## 6. Задати змінні середовища
```bash
export TELEGRAM_BOT_TOKEN="твій_токен"
export TELEGRAM_CHAT_ID="@digital_ranok"   # твій канал "Цифровий Ранок"
export GEMINI_API_KEY="AIza..."          # опційно
```

## 7. Запустити вручну (перевірка)
```bash
python news_bot.py
```

## 8. Автоматичний запуск за розкладом
### Варіант A: cron (якщо є свій сервер/комп'ютер, що завжди увімкнений)
```bash
crontab -e
```
Додай рядок (запуск щодня о 9:00 і 21:00):
```
0 9,21 * * * cd /шлях/до/папки && /usr/bin/python3 news_bot.py >> bot.log 2>&1
```

### Варіант B: GitHub Actions (безкоштовно, без свого сервера)
⚠️ **Важливо:** cron у GitHub Actions працює за **UTC**, але для розкладу "кожні 5 хвилин" це не має значення. Розклад `*/5 * * * *` вимагає **публічного репозиторію** (безлімітні безкоштовні Actions-хвилини); на приватному — вкладеться десь у межі, але може підбити 2000 безкоштовних хвилин/міс раніше.
1. Заведи приватний репозиторій, поклади туди `news_bot.py` і `requirements.txt`.
2. У Settings → Secrets додай `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`, `TELEGRAM_ADMIN_CHAT_ID`, `GEMINI_API_KEY`.
3. Створи файл `.github/workflows/digest.yml`:
```yaml
name: News Digest
on:
  schedule:
    - cron: '*/5 * * * *'  # кожні 5 хвилин (потребує публічного репо)
  workflow_dispatch:
jobs:
  run:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: python news_bot.py
        env:
          TELEGRAM_BOT_TOKEN: ${{ secrets.TELEGRAM_BOT_TOKEN }}
          TELEGRAM_CHAT_ID: ${{ secrets.TELEGRAM_CHAT_ID }}
          GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
```
GitHub сам запускатиме скрипт за розкладом — комп'ютер тримати увімкненим не треба.

## Мова публікацій
Джерела зараз — англомовні й українське (AIN.ua, англійська версія). Незалежно від мови оригіналу, Gemini перекладає і заголовок, і summary на українську перед публікацією — тобто в каналі завжди українською, хоч би що було в джерелі. Якщо додаватимеш нові RSS-джерела — це працює автоматично, спеціально нічого налаштовувати не треба.

## Що можна змінити
- Список RSS-джерел — у словнику `FEEDS` на початку `news_bot.py`.
- Кількість новин на категорію — `MAX_ITEMS_PER_CATEGORY`.
- Вікно часу (зараз 24 год) — `HOURS_WINDOW`.

## Перед тим як зробити репозиторій публічним
Секрети (`TELEGRAM_BOT_TOKEN`, `GEMINI_API_KEY` тощо) з GitHub Secrets публічними не стають — вони завжди приховані, незалежно від видимості репо.

Але якщо колись раніше ти комітив токени прямо в код (навіть якщо зараз їх там уже нема) — вони лишаються в **історії git**, і публічний репозиторій відкриє цю історію всім. Перед тим як перемикати видимість:
1. Перевипусти `TELEGRAM_BOT_TOKEN` через @BotFather → `/mybots` → бот → **API Token** → **Revoke current token**
2. Перевипусти `GEMINI_API_KEY` на aistudio.google.com/apikey (видали старий, створи новий)
3. Онови обидва secrets у GitHub → Settings → Secrets and variables → Actions
4. Тільки після цього: Settings репозиторію → внизу **Danger Zone** → **Change visibility** → **Make public**

## Команди для бота
Напиши боту `@digital_ranok_bot` в особисті (обов'язково спочатку `/start`, якщо ще не робив):

| Команда | Що робить |
|---|---|
| `/stats` або `/today` | Скільки постів і які саме за сьогодні |
| `/week` | Те саме за останні 7 днів |
| `/queue` | Скільки новин чекає в черзі на публікацію |
| `/sources` | Список активних RSS-джерел |
| `/pause` | Призупинити публікацію в канал (черга далі наповнюється, просто нічого не постить) |
| `/resume` | Відновити публікацію |
| `/help` | Список команд |

### Швидкість відповіді
Бот "прокидається" за розкладом `digest.yml` — зараз це **кожні 5 хвилин**, тож відповідь на команду приходить швидко, хоч і не миттєво (GitHub іноді запускає scheduled workflows з невеликою затримкою, кілька хвилин — це нормально, не помилка).

⚠️ Розклад раз на 5 хвилин вимагає, щоб **репозиторій був публічним** — інакше GitHub рахує це у платні хвилини (безкоштовно тільки 2000 хв/міс для приватних репо, а часті запуски цей ліміт швидко з'їдять).

**Перед тим як робити репо публічним:**
1. Перевір, що всі токени йдуть через `os.environ` в коді (жодних вписаних значень)
2. **Перевипусти токени**, якщо колись комітив їх у код (навіть якщо зараз видалив — старий коміт лишається в історії при публічному репо):
   - Telegram: @BotFather → `/mybots` → бот → **API Token** → **Revoke current token** → онови secret `TELEGRAM_BOT_TOKEN`
   - Gemini: aistudio.google.com/apikey → видали старий ключ, створи новий → онови secret `GEMINI_API_KEY`
3. У репозиторії: **Settings → General → Danger Zone → Change visibility → Make public**

## Захист від повторної публікації
Скрипт створює файл `posted_links.json` — список опублікованих новин (посилання, назва, категорія, час), пам'ятає 8 днів (щоб вистачило для `/week`). Перед публікацією він відфільтровує новини, які там уже є, тож при повторному запуску (вручну чи по розкладу) дублі не постяться.

Другий файл, `queue.json`, — черга новин, які вже "побачені", але ще не опубліковані (чекають своєї черги за темпом 1/годину). Новини старші за 48 годин з черги автоматично видаляються — застаріле не варто постити із запізненням.

Третій файл, `bot_state.json`, зберігає, з якого моменту бот вже прочитав твої команди, чи стоїть публікація на паузі (`/pause`), і коли була остання публікація (для темпу 1/годину).

## Режим "1 новина на годину"
Бот перевіряє нові новини й нові команди від тебе кожні 5 хвилин, але **публікує в канал рівно одну новину на годину**, не більше — навіть якщо за годину з'явилось 10 нових статей. Все зайве чекає в черзі (`queue.json`) і публікується по черзі: по колу між категоріями (IT/AI → Бізнес → Крипта → знову IT/AI...), так що жодна категорія не домінує.

Якщо новин накопичилось багато — перевір `/queue`, там видно скільки чекає і приблизно за скільки годин черга спорожніє.

Автоматичних звітів тобі в особисті немає — використовуй `/stats`, коли захочеш побачити, що опубліковано.

Файл автоматично зберігається назад у репозиторій після кожного запуску workflow (крок "Save posted links back to repo" в `digest.yml`). Тому:
- Права `permissions: contents: write` обов'язково мають бути в `digest.yml`
- Файли `posted_links.json`, `queue.json` і `bot_state.json` не треба створювати вручну — скрипт сам їх заведе при першому запуску
